# 📊 Strava Fitness Data Analysis

An end-to-end **Data Analytics project** that analyzes user fitness data from Strava to uncover insights related to **physical activity, sleep patterns, and health metrics such as BMI and weight**.

This project demonstrates the complete **data analytics workflow** including:

Data Cleaning → Exploratory Data Analysis → SQL Analysis → Interactive Dashboarding

---

# 🚀 Project Overview

Fitness tracking applications generate large amounts of user data including steps, calories burned, sleep patterns, and weight metrics. However, without structured analysis it becomes difficult to derive meaningful insights.

This project analyzes fitness data to identify **activity trends, sleep behavior, and health patterns** that can help organizations improve **user engagement and health monitoring strategies**.

---

# 🎯 Business Problem

Fitness platforms collect large volumes of user health data but lack clear insights into:

- How physical activity impacts calorie burn
- Whether sleep patterns influence fitness levels
- How user health metrics such as BMI and weight evolve over time
- Which activity behaviors are most common among users

Without these insights, companies cannot effectively improve **fitness recommendations or engagement strategies**.

---

# 💡 Business Objective

The objective of this project is to:

- Analyze daily activity trends
- Understand sleep duration and sleep quality
- Evaluate user health metrics such as weight and BMI
- Identify relationships between activity, sleep, and health
- Build dashboards that enable **data-driven decision making**

# 📈 Key Insights

Some important insights discovered during the analysis:

- Higher step counts are strongly associated with higher calorie burn
- User sleep duration remains relatively stable across most users
- Most users fall within the **Normal to Overweight BMI category**
- Activity levels vary significantly between users
- Weak correlation observed between steps and weight
